Categorisa de modelos de dados
	Modelos de dados conceituais(alto nível)
		Represenação para os usários especializados ou projetistas (modelo entidade-relacionamento)

	Modelos de dados lógicos (implementação)
		Nível intermediario de representação (modelo relacionado)

	Modelos de dados físicos (baixo nível)
		Representação para desenvolvedores de SGBDs (como estar realcionado / os comandos SKL que representão um modelo logico)


alto nível (não procedural) 
	permite a manipulação de conjunto de registros por ação (set-at-a-time)

baixo nível (procedural)
	permite a manipulação de um registro por ação (record-at-a-time), utiliza procedimento de linguagens de programação (script)

Os comandos DML podem ser embutidos em uma linguagem de propósito geral (host language) ou utilizada de modo interativo (query language)




   Esquemas são conjuntos de instâncias que representão elementos reais do banco de dados



iremos trabalhar com modo interativo não procedural
